The Wow Moment

One prompt. Claude builds a working invoice anomaly detector with a professional HTML dashboard.
It understands GST, duplicates, variance thresholds, and AUD formatting — all from the skill file, without being told.

Build me an invoice anomaly detector for our finance team. First create a realistic data/invoices.csv with about 30 invoices from 8 vendors — include some deliberate anomalies: a few duplicate invoices, some amounts significantly above that vendor's average, and a few missing PO numbers.

Then build a tool that:
1. Flags duplicate invoices (same vendor + similar amount within 7 days)
2. Flags amounts more than 30% above that vendor's average
3. Flags invoices missing a PO number
4. Generates a professional HTML report showing all flagged items with severity levels, the reason each was flagged, and the total risk exposure in dollars

Follow our project patterns and conventions.

Before hitting enter: "I haven't told it what GST is. I haven't told it to format in Australian dollars. I haven't told it our code style. It knows all of that from the CLAUDE.md and the skill file. One prompt. Let's go."

Prep Checklist
BEFORE RECORDING
  • Do a full dry run of the finale prompt — note timing, where Claude pauses, and where to narrate
  • Pre-install deps — npm install, prettier. No install output eating demo time
  • Git init with main branch, pre-create feat/demo branch for Act 2
  • Have a backup HTML report pre-generated — if Claude takes too long, open it and say "here's what it produced"
  • Terminal: 18pt+ font, dark theme, minimal prompt, large screen
  • Close Slack, email, notifications — clean recording
HOOKS NOTE
  • Only demo the Bash blocker in Act 2 — PreToolUse reliably blocks Bash commands (exit code 2)
  • Skip Edit/Write branch protection live — known upstream bug where PreToolUse can't reliably block file edits. Mention it conceptually ("you can also guard file edits") but don't demo it
  • PostToolUse auto-format is the real hero — visible during the finale when every file gets prettified
Q&A ANSWERS (10 min)
  • "Is it safe?" → Hooks enforce guardrails deterministically. You review every change like a PR.
  • "Can non-devs use it?" → The finale prompt was plain English. Slash commands abstract the rest.
  • "Sensitive data?" → Runs locally in your terminal. Code doesn't leave your machine.
  • "Cost?" → Anthropic Pro plan. The real question: how much does a dev-hour cost vs. what this saves?
  • "Replace our devs?" → Force multiplier, not replacement. Human decides what to build and reviews output.
  • "vs Copilot?" → Copilot suggests lines. Claude Code builds entire features. Different category.
