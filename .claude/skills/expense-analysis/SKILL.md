# Expense Analysis Skill

## CSV Format
Expense CSVs follow this schema:
```
date,description,amount,category,employee
```

- `date`: ISO format YYYY-MM-DD
- `description`: Free text, max 100 chars
- `amount`: Decimal, always positive (AUD)
- `category`: One of: Travel, Meals, Software, Office, Training, Marketing
- `employee`: Full name

## Categorisation Rules
When auto-categorising expenses:
- Uber, taxi, flights, hotels → **Travel**
- Restaurants, cafes, catering, lunch → **Meals**
- Subscriptions, licenses, SaaS → **Software**
- Stationery, furniture, equipment → **Office**
- Courses, conferences, books → **Training**
- Ads, campaigns, sponsorships → **Marketing**

## Australian Formatting
- Currency: `$X,XXX.XX` with AUD suffix where ambiguous
- Dates: Display as `DD/MM/YYYY` (Australian format)
- GST: 10% included in all amounts unless marked "GST-free"

## Report Structure
Every report must include:
1. **Header**: Report title, date range, generation timestamp
2. **Summary**: Grand total, category breakdown with percentages
3. **Detail**: Line items grouped by category, sorted by date
4. **Footer**: "Generated by FinanceBot — Digital Department"

## Validation Rules
- Reject negative amounts
- Reject future dates (beyond today)
- Flag amounts over $5,000 for review
- Warn if category is empty (default to "Uncategorised")
